# خطة عمل وتطوير: لوحة تحكم الإدارة العليا (Super Admin Dashboard) 🚀

بناءً على مراجعة ملفات التوثيق ([PROJECT_DOCUMENTATION.md](file:///f:/PROGRAMMING/matgarcoo/matgarco/PROJECT_DOCUMENTATION.md) و [PROJECT_PROGRESS.md](file:///f:/PROGRAMMING/matgarcoo/matgarco/PROJECT_PROGRESS.md))، منصة Matgarco هي منصة SaaS متعددة المتاجر. دور لوحة تحكم الإدارة العليا هنا هو **"عقل المنصة"**، حيث تمكنك كمالك من مراقبة صحة النظام، إدارة التجار، تتبع الأرباح، وقبل كل شيء: **التحكم الجذري واللامحدود بالقوالب والاشتراكات.**

فيما يلي خطة التطوير المفصلة والشاملة للمشروع الجديد (`super-admin-react`):

---

## 1. الهيكل المقترح للصفحات والأقسام (Sitemap & Features)

### 📊 أ. لوحة القيادة الشاملة (Super Overview / Dashboard)
* **مؤشرات الأداء الرئيسية اللحظية (Live KPIs):** إجمالي التجار، المتاجر النشطة، الإيرادات الشهرية المتكررة المنظمة (MRR)، والإيرادات السنوية (ARR)، بالإضافة لإجمالي المبيعات التي تمت على منصة Matgarco ككل (Platform Gross Merchandise Value).
* **رسوم بيانية ديناميكية:** معدل نمو التسجيلات الجديدة (يومي/أسبوعي/شهري)، نمو الإيرادات، توزيع التجار على باقات الاشتراك.
* **النبض اليومي (Daily Pulse):** تنبيهات سريعة (مثال: "3 متاجر جديدة سجلت اليوم"، "متجر X حقق مبيعات بـ 50 ألف").

### 🏪 ب. إدارة التجار والمتاجر (Merchants Management)
* **مركز بيانات التاجر (Merchant Core Data):** 
  * بيانات المالك، الخطة الحالية، الإيرادات المنفردة للمتجر، حجم الترافيك (الزوار).
* **أدوات تحكم فائقة (God Mode Tools):** 
  * إيقاف/تفعيل متجر مؤقتاً أو نهائياً (Suspend/Ban).
  * **تسجيل الدخول كتاجر (Impersonate Merchant):** الدخول بضغطة زر لحساب التاجر (بدون باسورد) لرؤية الإعدادات وحل المشاكل الفنية، واجهة Shopify تستخدم ذلك بكثرة.
  * إرسال رسائل أو تنبيهات فردية (Custom Notification) تظهر للتاجر في لوحة تحكمه الخاصة.

### 💳 ج. التقارير المالية والاشتراكات (Advanced Subscriptions & Billing Reports)
* **إدارة الخطط (Plans Engine):** التحكم في سعر كل خطة (Free, Starter, Pro, Business) والعمولة (Commission Rate)، والـ Limits الخاصة بها (عدد المنتجات، صلاحيات فريق العمل، رصيد الذكاء الاصطناعي).
* **تقارير مالية المتقدمة (Finance Analytics):**
  * **Chrun Rate (معدل التسرب):** نسبة التجار الذين ألغوا اشتراكهم هذا الشهر.
  * **LTV (القيمة الدائمة للعميل):** متوسط الأرباح المتوقعة من التاجر الواحد.
  * **إيرادات العمولات (Commission Revenue):** تقرير خاص بأرباحك من نسبة المبيعات (مثال: 2% من باقة Starter).
* **سجل الفواتير:** جدول عمليات دفع الاشتراكات (ناجحة/فاشلة/مستردة) مع إمكانية إصدار فواتير ضريبية يدوية.

### 💻 د. محرك القوالب الديناميكي المتقدم (Dynamic Theme Engine Manager) 🌟
*(هذا القسم هو أهم تعديل جوهري لتحويل القوالب من مجرد تغيير ألوان إلى معماريات متكاملة)*

* **نظام رفع القوالب (Theme Uploader / Store):** 
  * رفع قوالب جديدة للمنصة بالكامل في أي وقت (مثلاً ملف ZIP يحتوي على بناء Next.js Component مبني مسبقاً، مع ملف [config.json](file:///f:/PROGRAMMING/matgarcoo/matgarco/landing-next/tsconfig.json) يحدد الخيارات المتغيرة). 
  * *ملاحظة:* لتحقيق مرونة Shopify، سيكون القالب عبارة عن مكتبة مكونات (Component Library) يتم استدعاؤها بناءً على الـ JSON Config في قاعدة البيانات.
* **التحكم في وصول القوالب (Theme Access Control):**
  * تصنيف القوالب (ملابس، إلكترونيات، طعام، رقمي).
  * **ربط القالب بالاشتراكات (Tie to Tiers):** يمكنك تحديد قالب معين كـ "Premium" بحيث لا يظهر إلا لمشتركي باقة "Business" أو يتم عرضه للجميع ولكن مع علامة 👑 (قفل) لحث التجار على الترقية.
* **إدارة إصدارات القوالب (Theme Versioning):** نشر تحديث جديد لقالب (مثلاً إضافة قسم Testimonials للقالب الأساسي) وسيتم تحديثه تلقائياً أو بإشعار عند كافة المتاجر التي تستخدمه.

### ⚙️ هـ. إعدادات المنصة المركزية (Global Platform Settings)
* **بوابات الدفع المركزية (Payment Gateways):** 
  * إدخال الـ Webhooks والـ API Keys الخاصة بـ Paymob وغيرها، وتحويل الأموال للمنصة واستقطاع العمولة قبل تحويل الباقي للتاجر (Split Payments).
* **شركات الشحن المركزية (Global Shipping APIs):** إعداد حساب منصة Matgarco المركزي مع (بوسطة/أرامكس) ليستخدمه التجار أصحاب الباقات الأولى.
* **نظام التحديثات (Feature Flags):** إغلاق خاصية في المنصة (مثل الـ AI) عند كل التجار بضغطة زر في حالات الطوارئ والفحص.

### 📢 و. ميزات إضافية نوعية (Proposed Enhancements)
* **نظام التذاكر والدعم الفني المدمج (Support Ticket System):** 
  * يمكن للتاجر فتح تذكرة دعم من لوحته الخاصة "أواجه مشكلة في الشحن"، وتظهر التذكرة في لوحة الإدارة العليا ليقوم فريق الدعم بالرد عليها مباشرة.
* **إعلانات وإشعارات عامة (Global Announcements):**
  * إضافة بنر علوي (Top Banner) مؤقت في جميع لوحات تحكم التجار، مثال: "تم إطلاق قالب جديد اليوم 🔥 قم بالترقية للوصول إليه".
* **نظام الشركاء/المسوقين بالعمولة (Affiliate System for Matgarco):** 
  * لوحة لتسجيل شركاء التسويق، ومنحهم روابط إحالة، لتتبع عدد المتاجر التي سجلت من خلالهم وحساب نسبتهم تلقائياً (Growth Engine قوي جداً).

## 2. الصلاحيات وأنظمة الأمان (Security & Roles)
في لوحة الإدارة العليا، يتم إنشاء أكثر من دور للموظفين، لحماية الداتا:
1. `Super Admin`: يمتلك كل الصلاحيات (أنت فقط).
2. `Finance Manager`: يرى الأرباح والفواتير والاشتراكات، ولكن لا يمكنه وقف المتاجر أو الدخول لحساباتها.
3. `Support Agent`: يمكنه التسجيل كتاجر (Impersonate)، رؤية بيانات المتجر، الرد على التذاكر، ولكن لا يرى إيرادات المنصة العامة.
4. `Theme Developer / Content`: يرفع مقالات المدونات أو يرفع قوالب جديدة للمنصة.

## 3. خطة العمل والتأسيس (Implementation Plan)

### المرحلة 1: التأسيس المعماري ولوحة القيادة 🏗️
- تجهيز المشروع בـ (Vite + React + TS) وربط الـ Auth (Super Admin).
- برمجة صفحة الـ **Dashboard Overview** بالرسوم البيانية والمؤشرات اللحظية والإحصائيات الخاصة بالإيرادات (MRR / ARR).

### المرحلة 2: محركات الإدارة (التجار والاشتراكات) 👥
- برمجة قسم **التقارير المتقدمة** والاشتراكات.
- برمجة قائمة التجار بكل حالاتها (فلترة ذكية، ربط بـ Paymob Status).
- تفعيل "Login as Merchant" (يحتاج Endpoint في الباك إند يعطي الـ Super Admin التوكن الخاص بالتاجر).

### المرحلة 3: محرك القوالب الديناميكي والدعم الفني 🎨
- تأسيس قسم القوالب، ربطها بأنواع الاشتراكات (Free/Pro).
- *(ستحتاج هذه الخطوة لبعض التعديلات في الباك إند وفي الـ Storefront Next.js ليستطيع استقبال القوالب من قاعدة البيانات بدل أن تكون هارد كودد).*
- نظام التذاكر المدمج الـ Support Center.
- بناء نظام إرسال الإشعارات والبنرات العالمية.
